По умолчанию VidBee сохраняет объединённые видеозагрузки в MP4 и переключается на MKV, если кодеки несовместимы. Изменить выбор можно в разделе **Настройки → Загрузка в один клик → Контейнер**.

## Варианты контейнеров

| Вариант | Поведение yt-dlp | Когда выбирать |
|---|---|---|
| **Auto (MP4/MKV)** | `--merge-output-format mp4/mkv` — предпочитает MP4, переходит к MKV, если кодеки не помещаются в MP4. | По умолчанию. Максимальная совместимость без сбоев муксинга (HEVC + аудио высокого разрешения на bilibili, WebM-фрагменты YouTube и т. п.). |
| **MP4** | `--merge-output-format mp4 --remux-video mp4` | Нужен именно MP4 (наилучшая совместимость с QuickTime, iOS, соцсетями). Может завершиться ошибкой на потоках, кодеки которых не вмещаются в MP4 — при `Conversion failed` переключайтесь обратно на **Auto**. |
| **MKV** | `--merge-output-format mkv --remux-video mkv` | Самый «толерантный» контейнер: вмещает любой кодек, который выдаёт yt-dlp. |
| **WebM** | `--merge-output-format webm --remux-video webm` | Самые маленькие файлы для потоков VP9/AV1 + Opus с YouTube. Сорвётся, если удалённый источник только в AAC/H.264. |
| **Original (по умолчанию yt-dlp)** | Флаг `--merge-output-format` не передаётся. | Нужны встроенные значения yt-dlp (MKV при объединении, исходное расширение для одиночного потока). |

## Загрузки аудио

Выбор контейнера действует только на **видео**. Загрузки аудио в один клик по-прежнему используют контейнер, предпочитаемый платформой (обычно M4A / MP3 / Opus).

## Где находится настройка

- **Десктоп**: Настройки → Общие → карточка «Загрузка в один клик» → Контейнер.
- **Web**: страница «Настройки» → «Загрузка в один клик» → Контейнер.

Настройка сохраняется мгновенно и применяется как к ручным загрузкам в один клик, так и к загрузкам из RSS-подписок.

## Почему MP4 иногда оказывается MKV

Когда выбран **Auto** и сочетание кодеков источника не помещается в MP4, yt-dlp пишет MKV с исходными кодеками, чтобы не терять качество. Чтобы всё-таки получить MP4, выберите контейнер **MP4** — загрузка завершится с понятной ошибкой кодека, если контент нельзя смуксить, и не свалится молча в MKV.

Если нужен MP4 из не-MP4 источника без риска ошибки, остаётся только перекодирование (медленно и с потерями). VidBee сейчас экспонирует только remux; перекодирование намеренно не предлагается.
